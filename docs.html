Docs & replication

Replicate it in one command.

ICS Sentinel runs on the Python standard library alone — no installs, no API key, no hardware. Clone it, run the demo, and watch an AI SOC analyst work a simulated water-treatment plant under attack. Five minutes, start to finish.

$ python -m ics_sentinel.demo

01 · Quickstart

Five steps. Two are optional.

Click copy on any command. The demo never breaks without an API key — it falls back to deterministic [MOCK] triage.

1

Check your Python

You need Python 3.11+. That's the only requirement — the demo has 0 dependencies.

python3 --version
2

Clone the repo

Everything lives in github.com/jamilf/OT-AI.

git clone https://github.com/jamilf/OT-AI.git && cd OT-AI
3

Run the demo

Simulates the plant, injects 5 attacks, detects, maps to ATT&CK, triages, and prints a ranked incident report — in [MOCK] mode if no API key is set.

python -m ics_sentinel.demo
4

Turn on Claude optional

With an Anthropic API key the triage switches to [AI] — live analysis by Claude. rich adds pretty terminal output.

pip install anthropic rich
export ANTHROPIC_API_KEY=sk-ant-…

Verify with the test suite 75 tests

python -m pytest

03 · Documentation

How it works, piece by piece.

Click any topic to expand it.

The 5 detection rules detection
  • Write-source allowlist — only 10.0.0.11 (the engineering workstation) may write. Any other writer is an unauthorized command message (T0855).
  • Safe-range check — the pump setpoint (40002) must stay in 20–90. A write of 200 is flagged even from the authorized host — catching insiders and compromised workstations (T0836, T0831).
  • Scan-window rule — benign polling touches 6 register/unit points; more than 12 distinct points in one window means someone is mapping the plant (T0846). The demo's scan hits 160 points in ~4 seconds.
  • Structural validation — every frame is parsed against the Modbus spec; illegal function codes like 0x5A and 0x00 never reach the rules behind it (T0814).
  • Statistical rate baseline — per-source frame rate vs a mean + 3σ baseline learned from benign traffic catches the 40×/second replay flood (T0814, T0855).
Architecture: the 5-stage pipeline design

Traffic → Detection → ATT&CK mapping → AI triage → Report.

The default run narrows like a funnel: 387 frames → 10 alerts → 1 Critical incident → 1 executive summary. Each stage is a small, testable module: the simulator emits Modbus TCP frames, the detection engine raises alerts, every alert is tagged with MITRE ATT&CK for ICS techniques, Claude (or the mock) triages each one, and the reporter ranks them Critical-first with a single executive summary across the incident.

See the animated pipeline on the showcase page.

The simulated plant simulation

A water-treatment skid with five hosts:

  • 10.0.0.10 — HMI master: polls every 2s, never writes
  • 10.0.0.11 — Engineering workstation: the only authorized write source
  • 10.0.0.20 — PLC-1 (unit 1, intake)
  • 10.0.0.21 — PLC-2 (unit 2, storage)
  • 10.0.0.66 — Attacker: a compromised host on the OT segment

Registers: 40001 tank level (0–100%), 40002 pump setpoint (safe range 20–90), coil 0 pump on/off. Tank levels follow real pump physics (hysteresis control), so the baseline is physically coherent — not random noise.

AI triage & the mock fallback [AI] / [MOCK]

For every alert, the analyst produces: a severity with justification, a plain-English explanation an operator can act on, an attack narrative, ordered response actions, and a false-positive likelihood. With ANTHROPIC_API_KEY set, that's Claude; without it, a deterministic mock produces the same structure, so the demo never breaks offline.

Alert coalescing keeps the LLM bill sane: the 40×/second replay flood becomes 2 alerts, not 41 — roughly 20× fewer LLM calls.

Severity model & proof points quality

Default-run outcome: 1 Critical · 5 High · 4 Medium · 0 Low. The Critical is the process-safety violation (setpoint 200 vs safe max 90) — ranked first in the report.

0 false positives on benign traffic, tested across seeds and a 10-minute stream. 75 pytest tests pin every number, and the whole demo needs 0 dependencies beyond Python 3.11+.
