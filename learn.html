The 5 detection rules detection
  • Write-source allowlist — only 10.0.0.11 (the engineering workstation) may write. Any other writer is an unauthorized command message (T0855).
  • Safe-range check — the pump setpoint (40002) must stay in 20–90. A write of 200 is flagged even from the authorized host — catching insiders and compromised workstations (T0836, T0831).
  • Scan-window rule — benign polling touches 6 register/unit points; more than 12 distinct points in one window means someone is mapping the plant (T0846). The demo's scan hits 160 points in ~4 seconds.
  • Structural validation — every frame is parsed against the Modbus spec; illegal function codes like 0x5A and 0x00 never reach the rules behind it (T0814).
  • Statistical rate baseline — per-source frame rate vs a mean + 3σ baseline learned from benign traffic catches the 40×/second replay flood (T0814, T0855).

Why is any of this possible? Standard Modbus/TCP (port 502) predates security: no authentication, no encryption. Modbus/TCP Security (TLS on port 802) exists, but it is rarely deployed in brownfield plants — which is exactly why passive network detection matters.

Architecture: the 5-stage pipeline design

Traffic → Detection → ATT&CK mapping → AI triage → Report.

The default run narrows like a funnel: 387 frames → 10 alerts → 1 Critical incident → 1 executive summary. Each stage is a small, testable module: the simulator emits Modbus TCP frames, the detection engine raises alerts, every alert is tagged with MITRE ATT&CK for ICS techniques, Claude (or the mock) triages each one, and the reporter ranks them Critical-first with a single executive summary across the incident.

Monitoring is fully passive. Sentinel reads a SPAN/TAP-style mirror of the traffic — it never injects a packet, never polls a device, never touches the process. That matters in OT, where an active scanner can crash a fragile PLC.

The simulated plant simulation

A water-treatment skid with five hosts:

  • 10.0.0.10 — HMI master: polls every 2s, never writes
  • 10.0.0.11 — Engineering workstation: the only authorized write source
  • 10.0.0.20 — PLC-1 (unit 1, intake)
  • 10.0.0.21 — PLC-2 (unit 2, storage)
  • 10.0.0.66 — Attacker: a compromised host on the OT segment

Registers: 40001 tank level (0–100%), 40002 pump setpoint (safe range 20–90), coil 0 pump on/off. Tank levels follow real pump physics (hysteresis control), so the baseline is physically coherent — not random noise.

AI triage & the mock fallback [AI] / [MOCK]

For every alert, the analyst produces: a severity with justification, a plain-English explanation an operator can act on, an attack narrative, ordered response actions, and a false-positive likelihood. With ANTHROPIC_API_KEY set, that's Claude; without it, a deterministic mock produces the same structure, so the demo never breaks offline.

Alert coalescing keeps the LLM bill sane: the 40×/second replay flood becomes 2 alerts, not 41 — roughly 20× fewer LLM calls.

Severity model & proof points quality

Default-run outcome: 1 Critical · 5 High · 4 Medium · 0 Low. The Critical is the process-safety violation — the pump setpoint (expressed in %, safe range 20–90) forced to 200 — ranked first in the report.

0 false positives on benign traffic, tested across seeds and a 10-minute stream. 75 pytest tests pin every number, and the whole demo needs 0 dependencies beyond Python 3.11+.

02 · Explain it like I'm 5

Every term, in plain English.

Every word on this site, explained like you've never set foot in a plant. Search for a term or filter by category.

OT Plant floor

Operational Technology — the computers and controllers that run physical equipment: pumps, valves, turbines, assembly lines.

ELI5 If IT is the computers that handle your email, OT is the computers that turn the water on. Breaking IT loses data; breaking OT can flood a tank.

ICS Plant floor

Industrial Control System — the whole collection of OT gear that monitors and controls an industrial process.

ELI5 The "nervous system" of a factory or utility: sensors that feel, controllers that decide, and motors that act.

PLC Plant floor

Programmable Logic Controller — a rugged little computer that reads sensors and drives equipment by simple rules. No keyboard, no antivirus.

ELI5 A thermostat with a rulebook: "if the tank drops below 20%, turn the pump on." It does exactly what it's told — which is the whole problem when an attacker does the telling.

HMI Plant floor

Human-Machine Interface — the operator's screen showing tank levels, pump status, and alarms. In this demo it only reads.

ELI5 The dashboard in your car. It shows your speed; it doesn't press the pedal. So a "command" claiming to come from the dashboard is instantly fishy.

Engineering workstation Plant floor

The one trusted PC engineers use to reprogram or adjust PLCs — the only authorized write source on the network.

ELI5 The one set of car keys that's allowed to start the engine. Powerful — which is why, if someone steals it, Sentinel still checks what it's trying to do.

SCADA Plant floor

Supervisory Control and Data Acquisition — the wider system that gathers data from many PLCs/sites and lets operators supervise them centrally.

ELI5 Mission control for a plant: lots of screens pulling in readings from equipment that might be spread across a whole city.

Setpoint Plant floor

The target value a controller aims for — e.g. pump setpoint 60 means "run the pump at 60%". Safe range here is 20–90.

ELI5 The number you set on the oven dial. Set it to 200 when the safe max is 90 and something is going to burn — or in our case, overflow.

Register & coil Plant floor

The PLC's memory slots. A register holds a number (tank level, setpoint); a coil holds an on/off bit (pump running or not).

ELI5 Labeled boxes inside the controller. Box 40001 = how full the tank is. Box 40002 = how hard to run the pump. Coil 0 = pump switch.

Hysteresis Plant floor

Control that uses a gap between on/off thresholds so equipment doesn't rapidly flip. The demo's tank levels follow this real physics.

ELI5 Your home heating doesn't flick on and off every second — it waits until the room is a bit too cold, then a bit too warm. That "wait" keeps the pump from chattering, and makes the fake plant behave like a real one.

Modbus TCP Network

A 1979 industrial protocol (here over TCP port 502) for talking to PLCs. It has no authentication and no encryption.

ELI5 A walkie-talkie channel where every device obeys any voice it hears — no password, no caller ID. Whoever speaks, the machine listens.

Frame / packet Network

One message on the wire — e.g. "read register 40001" or "write 60 to 40002". The demo captured 387 of them in 60 seconds.

ELI5 A single envelope sent between machines. Sentinel reads every envelope going past and decides which ones look like trouble.

Trust boundary Network

The line separating devices that belong on the control network from those that don't. The attacker sits outside it.

ELI5 The fence around a playground. Being inside doesn't make you a kid — but anything that climbed the fence definitely shouldn't be there.

IP address Network

A device's address on the network, like 10.0.0.21. Sentinel uses them to tell who is talking to whom.

ELI5 A house number for a computer. 10.0.0.66 is the address Sentinel learns to distrust.

Unauthorized write Attacks

A command that changes a PLC, sent from a host that isn't allowed to write (anything but 10.0.0.11). Maps to ATT&CK T0855.

ELI5 A stranger walking up to the factory controls and flipping a switch. The command looks normal — it's who sent it that's wrong.

Recon scan Attacks

An attacker probing many registers/units to map the plant — 160 points in ~4s vs the 6 that benign polling touches. ATT&CK T0846.

ELI5 A burglar trying every door and window on the street to learn the layout. Normal residents only ever use their own front door.

Replay attack Attacks

Capturing a legitimate command and re-sending it over and over — here 40 times a second, byte-identical. ATT&CK T0814 + T0855.

ELI5 Recording someone saying "open the gate," then playing it on a loop 40 times a second. The words are real; the spam is the giveaway.

Malformed frames Attacks

Messages that break the Modbus rules — illegal function codes (0x5A, 0x00) or corrupt payloads, often used to crash fragile devices. ATT&CK T0814.

ELI5 Posting a ripped, gibberish-filled envelope through the slot hoping the machine chokes on it. Sentinel bins it before it can.

Dangerous setpoint Attacks

A valid-looking write that pushes a value out of its safe range — pump setpoint forced to 200 (safe max 90), even from the trusted host. ATT&CK T0836 + T0831.

ELI5 Someone with the real car keys flooring the accelerator to 200 mph. Right key, catastrophic intent — so we check the speed, not just the key.

MITRE ATT&CK for ICS Attacks

A public catalog of known adversary techniques against industrial systems, each with an ID (T0855, T0836…). Sentinel tags every alert with the matching ones.

ELI5 A field guide of known burglar tricks, each with a catalog number. Instead of "something weird happened," the report says exactly which trick this was.

Passive monitoring Defense

Watching a copy (SPAN/TAP mirror) of network traffic without sending anything yourself — so you can't disturb the process.

ELI5 A security camera, not a security guard who rattles the doors. It only watches — so it can never accidentally knock something over.

SOC Defense

Security Operations Center — the team (and tooling) that watches for attacks and responds. ICS Sentinel acts as an AI SOC analyst.

ELI5 The room full of people watching the security monitors. Sentinel is an AI sitting in one of those chairs.

Triage Defense

Sorting alerts by how serious they are and deciding what to do first — severity, explanation, and response actions for each.

ELI5 What the ER nurse does: the sickest patient gets seen first. Sentinel puts the tank-overflow alert ahead of the minor ones.

Alert & false positive Defense

An alert flags possible trouble. A false positive is an alert that turns out to be harmless — the noise that buries real threats.

ELI5 The smoke alarm going off because of toast. Too many and people start ignoring it — which is why "0 false positives" matters so much.

Baseline (mean + 3σ) Defense

Learn what "normal" traffic looks like (its average and spread), then flag anything far beyond it — here, 3 standard deviations above the mean rate.

ELI5 You know your commute takes about 20 minutes. Today it took an hour — way outside normal, so something's clearly wrong. That's the math Sentinel uses on traffic rates.

Allowlist Defense

A short list of who's explicitly permitted to do something (here, write to PLCs). Everyone not on it is denied by default.

ELI5 The bouncer's guest list. Your name's on it or you're not getting in — no debate.

LLM / Claude AI

A Large Language Model — AI that understands and writes text. ICS Sentinel uses Claude to turn raw alerts into plain-English analysis.

ELI5 A very well-read assistant. Hand it a cryptic alarm and it writes back, in normal words, what happened and what to do about it.

Mock fallback AI

A deterministic stand-in that produces the same report structure when no AI API key is set, so the demo always runs.

ELI5 A spare tyre. Not as fancy as the real wheel, but it means you're never stranded — the demo works even with the AI switched off.

Alert coalescing AI

Merging a flood of near-identical alerts into a few, so the AI isn't called 41 times for one replay attack — just twice.

ELI5 One text saying "he knocked 40 times" instead of 40 separate buzzes. Same story, far less noise — and a far smaller AI bill.
